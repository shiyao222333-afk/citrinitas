# 熔知 · 小白术语表

> 给看不懂代码的人（和用户未来的对话）看的。每个词用大白话解释。
> 代码里遇到不懂的词，先来这里查。详细注释也在各代码文件顶部的「小白导读」里。

- **知识库**：把所有文档存起来、能智能搜索的地方（底层是 Qdrant 数据库）。
- **入库 / 摄入（ingest）**：把一份文件（书、笔记、网页）放进知识库的过程。
- **向量 / 嵌入（vector / embedding）**：AI 把一段文字变成一串数字。意思相近的文字，数字也相近，这样就能"按意思搜"而不是只"按字面搜"。
- **分块（chunk）**：长文件会被切成小段，每段单独变成一串数字再入库（一段对应一个"点位"）。
- **点位 / 点（point）**：知识库里最小的一格——存着"一小段文字 + 它的数字向量 + 它的标签"。
- **受控词表（controlled vocabulary）**：一份"标准词清单"。AI 打的标签必须在这清单里，否则文档进待审核队列。
- **归一化（normalization）**：把各种写法统一成一种。例："齿轮设计"和"gear design"都归一成标准词"机械设计"。
- **分面（facet）**：给文档分类的几个固定维度，如"内容类型 / 领域 / 时效 / 可信度"。
- **枚举守卫**：给分类设的"固定选项表"，把 AI 写偏的词纠正回标准词。
- **待审核队列（review queue）**：AI 不太确定、需要人过目确认的文档集合。
- **置信度路由（confidence routing）**：AI 给自己打分，分高自动入库，分低进待审核。
- **守望夹（watcher）**：一个文件夹，丢文件进去会自动入库（不用手动点）。
- **死信（dead letter）**：处理失败、被挑出来单独处理的文件。
- **词表医生（vocab_doctor）**：一个清理脚本，把已入库旧文档里"不在词表"的标签，按现在的词表归并一遍。
- **doc_id**：每份文件的"身份证号"。同一份文件永远得到同一个号 → 重新放入=更新而非新增。
